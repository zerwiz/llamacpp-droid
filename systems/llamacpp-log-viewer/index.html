Local AI “swapping”

In local AI, swapping usually means:

  • Context shift — Move the KV cache (conversation memory) so you can keep talking past the context limit. Old tokens are dropped to make room for new ones (no restart).
  • KV cache type — Use quantized cache (e.g. q4_0) to fit more context in VRAM (less precision, more room).
  • Model swap — Switch between a heavy model (e.g. 30B coder) and a light one (e.g. 7B chat) so the GPU isn’t stuck on one model 24/7. Tools like Llama-swap (a proxy between your editor and the API) can swap models on demand; here you use presets to do it manually.

On each container form you can enable Context shift, Cache prompt, Cache reuse, and set KV cache K/V and Cache RAM. Below: quick presets to swap the same container between a heavy and a light setup.

Read online: llama.cpp server README — flags and defaults for your server version.
